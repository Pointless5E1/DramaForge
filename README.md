**NovelForge** 是一款具備數百萬字級長篇創作潛力的 AI 輔助寫作工具。它不僅是編輯器，更是一套集世界觀構建、結構化內容生成於一體的解決方案。

長篇創作中，維持一致性、保證可控性、激發持續靈感是最大的挑戰。爲此，NovelForge 圍繞四大核心理念構建：模塊化的 **“卡片”**、可自定義的 **“動態輸出模型”**、靈活的 **“上下文注入”** 與保證一致性的 **“知識圖譜”**。

<a id="核心特性"></a>
## ✨ 核心特性

*   **📚 Schema 驅動的卡片創作**
    *   每種卡片都可定義結構（Schema），AI 生成會按結構校驗，減少“看起來能用、落地卻混亂”的輸出。

*   **⚡ 指令流式 AI 卡片生成**
    *   不再是“一次性整段生成”。現在是“輸入要求 → 字段粒度流式填充 → 你確認或反饋繼續生成”，更可控、更容易修正。生成過程更加絲滑，避免長時間等待生成結果。
    *   該能力聚焦於“當前這張卡片”的生成與完善，關閉生成對話框後本次會話即結束。

*   **📝 章節正文字數控制**
    *   章節正文續寫支持兩種模式：`提示詞約束` 與 `控制模式`。
    *   `提示詞約束` 更自然、成本更低；`控制模式` 會按目標總字數切分多輪預算，控制更穩，但會消耗更多 token。

*   **✅ 通用審核與審核結果卡片**
    *   審核統一採用“草稿預覽 → 確認保存爲審核結果卡片”的流程。
    *   不同卡片類型可切換不同審核提示詞，但結果卡片結構保持一致，便於統一查看與引用。

*   **🧠 上下文注入 + 知識圖譜一致性**
    *   通過 `@DSL` 精準引用項目數據；結合 關係圖譜與動態信息，讓後續生成更貼近已寫內容與角色關係。

*   **🔮 靈感助手（Agent）**
    *   可持續對話、引用卡片、調用工具修改內容。你可以像和搭檔協作一樣打磨設定，而不是反覆整卡重生成。

*   **🧩 代碼式工作流系統**
    *   已重構爲代碼式工作流主線（去掉舊 DAG 方案），支持可視化編輯、觸發執行與複用，適合把常用創作流程自動化。

*   **🤖 工作流 Agent**
    *   你可以直接用自然語言描述需求，讓 Agent 幫你寫/改工作流代碼、做校驗並應用變更。

*   **💡 靈感工作臺 (Ideas Workbench)**
    *   支持自由卡片、跨項目引用、移動/複製回正式項目，適合專門做腦暴和素材沉澱。

- 🚀 **重大更新：0.9.0**

- ✨ **重構AI 卡片生成流程**
  - 從“點擊後等待整段結果”升級爲“輸入要求 → 對話框內字段粒度生成 → 確認/反饋繼續生成”，顯著增強可用性，更加絲滑~
  - 生成過程更可控，修改成本更低。

- 🧱 **工作流系統重構（探索性）**
  - 我們探索性地將工作流從舊的 **DAG 式編輯器** 遷移到新的 **代碼式工作流（Python 風格語句 + 特殊標記 DSL）**，並逐步移除了舊的 DAG 方案。
  - 目前更多是基於對可維護性與 AI 友好程度的綜合權衡。
  - **代碼式工作流的優點（當前體感）：**
    - 邏輯更線性、更清晰：順序、等待（`Logic.Wait`）、異步（`async=true`）等語義更貼近真實執行過程。
    - 進度處理與異步操作更自然：執行器可按語句計劃調度，不需要在圖上繞來繞去。
    - 對 AI 更友好：同一個功能，代碼式往往幾十行就能表達；而 DAG 配置經常需要幾百行的節點與連線描述。
  - **代碼式工作流的缺點（需要持續打磨）：**
    - 不如 DAG 直觀
    - 對字符串/代碼格式更敏感：參數序列化、字典字段類型、變量引用等細節更容易引發校驗或運行錯誤，需要更強的校驗與提示詞約束。

- 🤖 **新增工作流 Agent**
  - 可通過自然語言描述目標，由 Agent 生成/修改工作流代碼並做校驗。
  - 支持“先預覽再應用”的安全變更體驗。
  - 可能還有些bug

- 📚 **內置工作流增強**
  - 增加“拆書工作流”等實用流程模板，便於開箱使用和二次改造。

- 🎨 **靈感助手 UI 與交互優化**
  - 對話渲染、輸入區交互、工具調用顯示等體驗優化。

- 🧹 **工程重構與穩定性提升**
  - 前後端目錄結構和模塊邊界大幅改動、整理，代碼可維護性提升。
  - 修復一批工作流、可視化參數編輯、Agent 交互相關問題。

### 0. Neo4j Desktop（可選，非必須）

### 方式一：從源碼運行 (開發者/最新功能)（非開發者建議用方式二）

```bash
# 克隆倉庫
```


**3. 一行命令同時啓動前後端（npm）**
```bash
npm run dev
```

#### 重要：.env 的 BOOTSTRAP_OVERWRITE

> 啓動後端時，系統會按需初始化/更新內置資源（知識庫、提示詞、工作流等）。是否覆蓋更新由 `.env` 中的 `BOOTSTRAP_OVERWRITE` 控制。

- 建議設置：
  - 如果你沒有直接修改過內置資源，建議設置爲：
    ```ini
    BOOTSTRAP_OVERWRITE=true
    ```
    這樣可以在升級版本或重啓時自動同步最新的內置知識庫/提示詞/工作流。
  - 如果你曾直接修改過“內置”資源，建議設置爲 `false`，以避免被覆蓋。

- 建議（避免被覆蓋）：
  - 不要直接編輯“內置”資源。
  - 如需定製，請新建一個副本（複製知識庫/提示詞/工作流後重命名），在副本上修改。這樣即使將來設置 `BOOTSTRAP_OVERWRITE=true`，你的自定義副本也不會被更新邏輯覆蓋。

6.  返回上一級，運行 `NovelForge.exe` 啓動主程序。

> 大部分數據都存儲在backend/novelforge.db數據庫中，當版本更新/遷移時，將該數據庫文件複製到對應位置即可。

## ✍️ 創作流程

1.  **配置大語言模型 (LLM)**
    *   首次啓動後，在設置中添加你的 AI 模型配置，如 API Key、Base URL 等。
    ![alt text](docImgs/README/image.png)
    推薦使用Gemini 2.5Pro級別以上的LLM進行創作

2.  **創建項目與初始化工作流**
    *   新建項目時，可以選擇一個初始化工作流（通常是 `onprojectcreate` 類型）來自動創建預設卡片。系統內置了"項目創建·雪花創作法"工作流，會按照雪花創作法自動創建一套完整的卡片樹。
    ![alt text](docImgs/README/image-1.png)

3.  **自頂向下，填充核心設定**

    *   每張卡片都可以打開 AI 生成對話框，輸入本次要求，系統會按字段粒度流式生成。
    *   你可以在生成後選擇“確認”直接落庫，或提交反饋意見繼續迭代，不滿意不必整卡重來。
    完成核心藍圖卡片創作後，點擊保存，會自動根據分卷數量創建對應分卷卡片。
    繼續完成分卷大綱創作即可，從第1捲開始。
    完成之後，會自動根據階段數創建階段大綱子卡片、寫作指南卡片。建議先生成寫作指南卡片，生成寫作指導信息，再進行階段大綱卡片創作。

    生成完成後點擊完成，再保存卡片即可。或者對某些字段不滿意，則輸入指導意見進行反饋。
  

4.  **藉助靈感助手完善內容**
    *   在寫作過程中，如果你想進一步打磨或優化卡片內容，可以隨時使用右側的靈感助手。
    *   選中任意卡片後，靈感助手會自動讀取該卡片的內容，方便你參考和思考。
    *   你可以直接向助手提出具體問題，比如“這個角色的動機是否合理？”、“怎樣讓這個場景更有張力？”等。
    *   靈感助手會結合當前卡片內容，給出針對性的建議，你可以與助手反覆交流，逐步完善想法。
    *   通過“添加引用”按鈕，還能把當前項目或其他項目的相關卡片內容加入對話，激發更多創意火花。
    *   靈感助手具備感知上下文、調用工具修改/創建卡片內容的能力（實驗性）
    ![Alt text](docImgs/README/image-20.png)

#### AI 生成對話框 vs 靈感助手（如何選擇）

- **AI 生成對話框**：聚焦當前單張卡片，用於快速生成與迭代該卡片內容；會話僅持續到本次生成流程結束，關閉對話框後會話清空。
- **靈感助手**：用於跨卡片、跨項目的持續對話與創作；可引用多個卡片進行分析與聯動創作，並且對話歷史可持久保存。
- **建議用法**：
  - 目標是“把這張卡片寫好” → 用 AI 生成對話框。
  - 目標是“跨設定聯動思考/長期討論/多卡片協作” → 用靈感助手。

5.  **完成階段大綱創作後，自動生成章節大綱、章節正文卡片，並自動注入每章需要參與的實體。**

### 章節審核與通用審核

除章節正文外，其它卡片（例如階段大綱、通用文本等）也可以直接使用頂部的 **審核** 按鈕。

<a id="prompt-workshop"></a>
### 提示詞工坊 (Prompt Workshop)

*   所有 AI 功能的背後都是可編輯的提示詞模板。你可以在這裏修改預設模板，或創建全新的模板。
*   **知識庫注入**: 支持通過 `@KB{name=知識庫名稱}` 語法，在提示詞中動態引用"知識庫"內容，爲 AI 提供更豐富的背景信息。

<a id="context-dsl"></a>
### 上下文注入 (@DSL) 詳解

這是 NovelForge 的特色。它允許你在提示詞模板中，用 `@` 符號精確地引用項目中的任何數據注入爲上下文。

*   **按標題引用**: `@卡片標題` 或 `@卡片標題.content.某個字段`
*   **按類型引用**: `@type:角色卡` (所有角色卡)
*   **特殊引用**: `@self` (當前卡片), `@parent` (父卡片)
*   **強大的過濾器**:
    *   `[previous]`: 獲取同級的前一個卡片。
    *   `[previous:global:n]`: 獲取全局順序（樹狀先序）中最近的n個同類型卡片。
    *   `[sibling]`: 獲取所有同級兄弟卡片。
    *   `[index=...]`: 按序號獲取，支持表達式，如 `$self.content.volume_number - 1`。
    *   `[filter:...]`: 按條件過濾，如 `[filter:content.level > 5]` 或 `[filter:content.name in $self.content.entity_list]`。
*   **字段級別選中**: 可選中整個卡片數據，也可以單獨選中卡片的字段。

- 說明：這部分能力已可用，複雜流程下可能存在少量邊界問題（如個別恢復場景）。

<a id="workflow-persistent-vs-temporary"></a>
#### 持久化工作流 vs 臨時工作流

- **臨時工作流（默認）**：運行記錄用於當前查看與調試，後續會被自動清理。
- **持久化工作流**：開啓“持久化保存”後，運行記錄會長期保留（受系統保留策略影響）。


<a id="workflow-builtins"></a>
#### 內置工作流模板

系統預置了多個常用工作流，可直接使用或作爲參考：

- **項目創建·雪花創作法**: 新建項目時按照雪花創作法自動創建初始卡片結構
- **世界觀·轉組織**: 從世界觀設定的勢力列表自動生成組織卡
- **核心藍圖·落子卡**: 根據藍圖內容自動創建角色卡、場景卡和分卷卡片
- **分卷大綱·落子卡**: 根據分卷大綱自動創建階段大綱和寫作指南
- **階段大綱·落章節卡**: 根據階段大綱的章節列表自動創建章節大綱和正文卡片
- **拆書工作流**: 用於拆解既有文本結構並落地到卡片體系

<a id="workflow-project-init"></a>
#### 項目初始化工作流

新建項目時，可以選擇一個 `onprojectcreate` 觸發器的工作流作爲項目模板：

- 默認選擇"項目創建·雪花創作法"，自動創建作品標籤、金手指、一句話梗概、故事大綱、世界觀設定、核心藍圖等卡片
- 也可以在工作流工作室中創建自己的項目初始化工作流，完全自定義項目起始結構
- 支持複雜的初始化邏輯，如根據條件創建不同的卡片結構
