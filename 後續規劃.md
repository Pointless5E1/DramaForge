# 後續規劃

本文件用於記錄項目的短期 / 中期 / 長期規劃。

- 可能會有較多規劃項，也可能暫時爲空。
- 更新頻率不固定（按實際情況不定時更新）。
- 當本文件出現新的規劃內容時，通常意味着這些方向在短期內**不一定**會由作者立即發佈版本實現，但會先公開出來，方便有興趣的貢獻者參與。

## 貢獻協作約定

如果你對某個規劃功能感興趣並準備實現，建議先提交一個 Issue 說明：

1. 你準備實現哪一個規劃功能；
2. 你的實現思路（簡要）；


這樣可以減少與其他貢獻者的重複開發或衝突。

當該功能完成、PR 提交併被接收後，請關閉對應 Issue。

### Issue 標題格式建議

爲方便快速識別“規劃認領”與普通問題，建議使用以下中文標題格式：

- 認領規劃項：`【功能認領】【模塊】功能名稱`
- 方案討論：`【方案討論】【模塊】方案主題`

示例：

- `【功能認領】【工作流】工作流可視化調試面板增強`
- `【方案討論】【AI生成】指令流多輪反饋的狀態機優化`

建議在 Issue 正文附上：

1. 計劃實現範圍（做什麼 / 不做什麼）
2. 預期時間（可選）
3. 是否需要協作
